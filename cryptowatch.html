Cryptowatch trade ticket redesign

2020 · Research & User Interface Design

Cryptowatch is a charting and trading terminal for cryptocurrency markets, providing an interface to scan prices, analyze market movements, and make trades on most major exchanges. Instead of trading on five separate exchanges because of liquidity or trading pairs unavailable on certain exchanges, you can connect all your exchanges via API and have a one-stop shop for everything.

This case study is a personal project of mine to explore and visualize a new trade interface for Cryptowatch that mainly follows the usability heuristic recognition over recall.

Introduction

Working in the crypto industry for the past couple of years has taught me a lot about design. Money is a commodity, in one form or another. As someone who has lived in a first-world country and a developing one, being able to send and receive payments cross-border seamlessly has been one of the things that I like most about crypto.

Current design with opportunity for improvements

With Cryptowatch, traders can track market movements and directly make trades on all exchanges they've connected to the platform, saving more time and money. Since I'm more of an investor than a trader, and having used a few exchanges to trade crypto myself, my first impression was how Cryptowatch trade ticket UI is a little unrecognizable compared to most of the exchanges I've used, and perhaps it can use a few improvements to promote initializing and completing a trade.

Label above fields

Research about label placement using eyetracking shows that placing label over input field allows users to capture both elements within a single eye movement because of the proximity, while left-aligned labels actually force users to unnecessarily take more time to interact visually with the form.

  • Helps maintain a close and distinct visual relationship between the label and the form control.
  • Makes the form easier to scan, because users can see the text field in the same fixation as the label.
  • Establishes a mental model that the user is familiar with using the trade ticket on other exchanges.

Important fields as main focus

Order type, leverage, and expiry are settings that the user is likely to look at and set once and then move on, so these could use a hollow style dropdown so that there is less visual prominence and the user can focus on the most important fields.

Choose maximum amount with one click

  • Choose the maximum amount of crypto the user can trade based on available funds on the connected exchange.
  • This replaces the % capital options and the "Funds" field which isn't editable.
  • This would reduce clutter while still giving the user an option to see how much they can trade.

Editable "Total" amount

  • Allows more control for the user to enter their own number, “Quantity” will be automatically calculated.
  • User won’t need to guess or delay trade to calculate the math.
  • Two users (a while ago and recently) on Reddit commented they would love being able to use this feature.

Improved order confirmation step

  • Labels and values are displayed inline with line separators making it easier to scan the content.
  • "Back" button replaces "Cancel" and is position on the top left to avoid accidental clicks.
  • Current behavior on Cryptowatch is pressing "Cancel" doesn't erase anything and just directs the users back to the order form, labeling the button "Back" makes more sense in this context because the intent is to allow the user to edit the order.

Expanded trade ticket with advanced options

The ticket gradually gets increased in size as advanced options are toggled, which is a downside of having labels above fields.

Trade-offs and next steps

Removed "Reset" button: the goal is to encourage trading activities, and having two buttons at the bottom of a form clutters up the interface.

  • The extra choice also can slow the user down because it requires extra thinking, and the user can always erase or edit the fields.
  • "Reset" would be useful if the data to be entered differs significantly from what was entered before.
  • Run a test or refer to user testing to see what is the chance of user filling out everything and A to Z and then decide to start everything over, versus editing one or two fields.

Left label text for input fields on mobile to allow more space for trade call to action as mobile screen size is arbitrary. On mobile, we are losing even more space due to order book and orders/trades/positions panel which are non-negotiable components. Doing this, however, creates inconsistency between web on mobile.

These designs suggestions again are merely from my own observations and knowledge as a designer and a user. Actual Cryptowatch users might make a lot of complex trades and there are things behind the scene that I wouldn’t know unless I'm within the Cryptowatch team to have access to unique user insights and behaviors, and I'm sure there are.

A couple of data points from analytics to determine this design's performance — to see if it works as intended — could be the time spent on the ticket to finish a trade, and the amount of users doing more trades on the new ticket versus the old ticket via A/B testing.
